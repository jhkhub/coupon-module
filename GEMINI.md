**1. 비즈니스 시나리오 (Business Scenario)**
- **상황:** "치킨 2만원 할인 쿠폰 100개 선착순 발급!" 이벤트 진행.
- **현상:** 이벤트 시작 직후 수만 명의 사용자가 동시에 '발급' 버튼을 클릭.
- **목표:** 폭발적인 트래픽 속에서도 **정확히 100명**에게만 **1인 1매**씩 쿠폰을 지급하고 시스템을 안정적으로 유지함.
****

**2. 주요 기능 및 요구사항 (Requirements)**

| **구분** | **상세 내용**                                 |
| --- |-------------------------------------------|
| **정확성** | 어떠한 상황에서도 발행된 쿠폰 합계는 100개를 초과할 수 없음.      |
| **동시성** | 수천 ㅈ개의 스레드가 동시에 수량을 수정하려 할 때 데이터 꼬임 방지.   |
| **중복 방지** | 동일 사용자가 찰나의 순간에 여러 번 요청해도 반드시 1개만 발급.     |
| **안정성** | 대량의 트래픽 유입 시 DB 커넥션 고갈이나 서버 다운 없이 안정적 응답. |

**3. 목표**

- **결함 재현:** 동시성 제어가 없는 상태에서 '수량 초과 발급' 에러가 발생하는 것을 테스트 코드로 직접 확인.
- **문제 해결:** 발견된 결함을 해결하고, 멀티스레드 환경에서도 테스트가 성공하도록 구현.
- **과정 기록:** 발생한 문제의 원인과 해결 방법(비포/애프터)을 문서로 작성.
